# 서울 따릉이 이용량 기상 데이터 분석

> 기상 데이터(기온·강수량·미세먼지)로 서울 공공자전거(따릉이) 월별 이용량을 분석하고 예측 모델을 검증한 개인 프로젝트입니다.

**개발 기간**: 2026.04
**개발 환경**: Python 3.11 / Jupyter Notebook / VS Code

---

## ▶ 핵심 요약

기상 변수로 따릉이 이용량을 예측하면서, **단순 학습 R²(0.987)이 시계열 교차검증으로 평가하면 +0.418로 크게 떨어진다**는 것을 발견했습니다. 또한 미세먼지(PM2.5)를 변수로 추가했더니 **상관계수는 -0.629로 강했지만 CV 성능은 오히려 악화(+0.418 → +0.356)** 되었습니다. 원인은 기온과 PM2.5 사이의 다중공선성(r=-0.749)입니다.

> **이 프로젝트의 진짜 결과는 모델 정확도가 아니라, 모델 검증 과정에서 가설이 뒤집힌 사례입니다.**

## ▶ 분석 과정

### 1. 데이터 전처리
- CSV 13개 파일 병합 (`pd.concat`)
- 컬럼명 불일치 통일 (`대여년월` → `대여일자`, `이용거리` → `이동거리`)
- 날짜 형식 통일 (YYYYMM)
- 미세먼지 데이터: 측정소별 → 서울 월평균으로 집계
- 따릉이 + 기온 + 강수량 + 미세먼지 inner join

### 2. 탐색적 데이터 분석 (EDA)
- 월별 이용건수 추이
- 기온 / 강수량 / PM2.5 vs 이용건수 산점도
- 변수 간 상관행렬 (다중공선성 진단)

### 3. 예측 모델 비교
시계열 교차검증(TimeSeriesSplit, n_splits=5)으로 5가지 모델 조합을 비교했습니다.

## ▶ 핵심 인사이트

1. **기온이 압도적인 핵심 변수** — 단일 변수로 이용량 변동의 약 92% 설명
2. **PM2.5는 강한 상관(-0.629)을 보이지만 모델 성능 향상에 기여하지 못함** — 기온과의 다중공선성(r=-0.749) 때문에 기온이 이미 담고 있는 정보를 중복 제공
3. **학습 R²(0.987)와 시계열 CV R²(0.418)의 큰 격차** — 표본 크기가 작을 때(n=30) RandomForest는 학습 데이터를 외우는 경향이 있으며, 정직한 평가를 위해 시계열 CV 필수
4. **선형 모델은 이 문제에 부적합** — 기온-이용량 관계가 비선형(영하/극서기 모두 감소)이므로 RF가 우월
5. **변수 추가 ≠ 성능 향상** — 새 변수가 기존 변수와 상관이 높으면 노이즈만 추가될 수 있음
